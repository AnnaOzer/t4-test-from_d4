в папке t4/framework/4Mvc/Extensions
расширение это библиотека внутри фреймворка
это библиотеки для вью вроде бутстрапа и капчи и джиквери
это попытка интегрировать хорошо известные библиотеки для вью внутрь фреймворка

внутри папки расширения есть файл Extension.php
это класс расширения

этот класс должени реализовать всего один метод init()
обычно в этом методе публикуются ресурсы
например бутстрап, метод инит
    вначале получает конфиг расширения
    потом публикует ресурсы (методы publish, publishCssFile, publishJsFile)
        чтобы js и css файлы вставились в шаблон
        при просмотре кода страницы видим, что эти css и js файлы вставились

т.е. расширение - это библиотека внутри фреймворка,
библиотека, которая содержит в себе
    некие ресурсы
    и класс, управляющий этими ресурсами, у которого вызывается метод инит
и понятно теперь,
что t4/framework/Mvc/Application.php в методе initExtensions
из конфига перебирает все расширения,
    для каждого расширения создает объект и вызывает у него метод инит

оставлена лазейка для создания своих собственных расширений
(пока Альберт не видел случаев чтобы это как-то использовалось, хотя Василий Зиновьев пробовал)

Проблема: при таком подходе не получается следить за свежестью версий этих библиотек
(бутстрап и джейквери представлены не последними версиями)
